# Track Specification: 增强型相框交互与手势控制

## 概述 (Overview)
本项目旨在提升圣诞树应用中相框照片的交互体验。核心目标是实现一种“照片对齐切换”模式，使用户能够通过特定的手势（食指与拇指呈 L 形组合）在 3D 场景中快速、平滑地切换浏览不同相框，并让树体自动旋转对齐到当前选中的相框。

## 功能需求 (Functional Requirements)
1.  **照片中心对齐 (Center-Aligned Switching)**：
    *   在 3D 场景中，当切换到特定照片时，圣诞树应平滑旋转，使对应的相框正对着摄像机（屏幕中心）。
    *   旋转过程应具有平滑的插值（Lerp）动画效果。

2.  **L 形手势识别 (L-Shape Gesture Recognition)**：
    *   利用 MediaPipe Hands 识别特定的“L 形/八字形”手势（食指伸直，拇指伸直且与食指保持一定开角，其余手指握拢）。
    *   该手势作为“照片导航模式”的激活开关。

3.  **垂直位移控制 (Vertical Displacement Control)**：
    *   在保持 L 形手势的情况下，监测手部在 Y 轴方向的移动。
    *   **向上移动**：触发切换到“上一张”照片，树体顺时针/逆时针旋转至上一相框。
    *   **向下移动**：触发切换到“下一张”照片，树体顺时针/逆时针旋转至下一相框。
    *   需要设置合理的位移阈值，避免微小抖动误触发。

4.  **UI/反馈增强**：
    *   当识别到 L 形手势并准备切换时，给予用户视觉反馈（如手势追踪点变色或相框高亮）。
    *   支持循环切换（最后一张照片后回到第一张）。

## 非功能需求 (Non-Functional Requirements)
*   **低延迟**：手势识别到动作触发的延迟应小于 100ms。
*   **鲁棒性**：应能区分常规旋转手势（全手掌或抓取）与特定的 L 形导航手势。

## 验收标准 (Acceptance Criteria)
*   [ ] 成功识别 L 形手势，且不与现有的“握拳”或“胜利”手势冲突。
*   [ ] 向上/向下快速移动 L 形手势能准确触发照片切换。
*   [ ] 切换照片时，圣诞树能自动且平滑地旋转至该相框的正前方。
*   [ ] 在断开手势后，用户仍能通过现有的普通旋转/缩放手势控制圣诞树。

## 超出范围 (Out of Scope)
*   点击相框进入全屏“专注模式”的大图查看功能（本次仅关注 3D 场景内的联动）。
*   修改现有的圣诞树配色或基本粒子系统。
